# Doctor Notification and Prescription Search Validation

Date: 2026-06-10
Application: TOTTECH Clinical Services
Environment: /opt/tottech-one

## Summary

Implemented and validated the two P0 fixes.

- Pharmacy Master driven medicine autocomplete now starts after 3 characters in the Doctor Consultation prescription builder.
- Search supports medicine name, generic name, brand name, strength, dosage form, and medicine/strength combination matching.
- Selecting a medicine from Pharmacy Master populates medicine name, strength, and dosage form.
- Saved prescriptions now retain Pharmacy Master references through medicine_master_id / medicine_id.
- WhatsApp notification doctor_name mapping now resolves the real appointment doctor.
- Patient timeline prescription events now show the actual doctor name.

## Code Changes

- app/clinical-services/doctors/consultation/[id]/page.tsx
  - Added autocomplete prescription medicine input.
  - Added Pharmacy Master-backed medicine selection.
  - Added dosage form display/edit field.
  - Added hidden/reference fields medicine_master_id and medicine_id in prescription row state.

- app/api/clinical/doctors/consultations/[id]/route.ts
  - Enriches posted medications from clinical_medicine_master.
  - Persists medicine_master_id, medicine_id, strength, dosage_form, generic_name, brand_name, and available stock in prescription JSON.
  - Joins doctors in the POST appointment loader so notification/timeline code has the actual doctor name.
  - Writes prescription timeline event with the real doctor name.

- lib/clinical/notification-service.ts
  - Centralized doctor-name fallback resolution.
  - Treats missing, generic Doctor, Doctor #N, and Assigned Doctor values as unresolved.
  - Resolves the actual doctor from appointment/patient context before queueing WhatsApp messages.
