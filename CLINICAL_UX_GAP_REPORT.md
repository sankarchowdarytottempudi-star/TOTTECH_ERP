# Clinical UX Gap Report

## Problem Summary
The clinical application exposed too many ERP-style navigation paths and operational users could still reach broad areas that were not part of their daily workflow.

## Gaps Addressed in This Sprint
- Role-based navigation was too broad for operational users.
- Analytics and technical domains were still visible to non-governance roles.
- The clinical dashboard showed generic quick actions that duplicated workflow navigation.
- The global clinical search field had visual overlap issues between the search icon and typed text.
- The main student growth trend chart could fail to render responsively because it depended on a measured width.

## Remaining UX Expectations
- Reception, Doctor, Lab, Pharmacy, Nurse, OT, ICU, IVF, and Admin should only see relevant workflow paths.
- Operational users should start from work queues, not module catalogs.
- The global search bar should remain easy to use and visually clear.
- Analytics should be reserved for governance roles.
